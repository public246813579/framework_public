# Thinking Framework Library

Thinking Framework Library は、このリポジトリ内に保存されている思考フレームワーク系HTMLファイルを紹介し、目的別に探せるようにした GitHub Pages 向けの静的ポータルサイトです。

- バックエンド不要
- 外部API不要
- npmビルド不要
- 外部フォント、CDN、第三者画像素材なし
- HTML / CSS / JavaScript のみで動作

## サイト構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | ポータルのトップページです。Hero、Quick Selector、カード一覧、おすすめ順、著作権・免責表示を配置しています。 |
| `style.css` | ダークモード風のミニマルなUI、カードレイアウト、検索バー、スマホ対応を定義しています。 |
| `app.js` | フレームワーク情報の配列、検索、カテゴリフィルター、用途別フィルター、図解カード自動生成を担当します。 |
| `preview*.html` | 既存の各フレームワーク集本体ページです。トップページからリンクします。 |

## GitHub Pages 公開手順

1. 変更をGitHubへ push します。
2. GitHubリポジトリの **Settings** を開きます。
3. **Pages** を選択します。
4. **Build and deployment** の Source で **Deploy from a branch** を選択します。
5. Branch に公開したいブランチを指定し、フォルダはリポジトリ直下の場合 `/ (root)` を選択します。
6. **Save** を押します。
7. 数十秒から数分後、表示される GitHub Pages URL にアクセスします。

このサイトはビルド工程を必要としないため、`index.html`、`style.css`、`app.js`、既存HTMLが同じ公開ディレクトリにあればそのまま動きます。

## 新しいHTMLファイルの追加方法

1. 新しいHTMLファイルをリポジトリへ追加します。
2. `app.js` の `frameworks` 配列へ1件追加します。
3. 以下の項目を設定します。
   - `id`: 一意のID
   - `file`: 実際のHTMLファイル名
   - `title`: カードに表示するタイトル
   - `sourceTitle`: 元HTMLのtitle相当
   - `category`: カテゴリ
   - `count`: 収録数
   - `scenes`: 使える場面
   - `features`: 主な機能
   - `summary`: 短い紹介文
   - `diagram`: 図解ノード。各ノードに短いラベルと実務で使うための説明を入れます。
   - `application`: 実務での具体的な使い方
4. ファイル名にスペースや括弧がある場合でも、リンク生成時に `encodeURI()` を使うためブラウザ上ではURLエンコードされます。

将来的には、公開URLの可読性と保守性のために `frameworks/essential-thinking.html` のような英数字・ハイフン中心のファイル名へリネームすることを推奨します。ただし、既存リンク互換性が必要な場合はリダイレクト用ページやリンク更新方針を検討してください。

## 既存HTMLの扱い

既存HTMLは各フレームワーク集の本体ページとして扱い、`index.html` からリンクしています。現在は利用者が迷子にならないよう、各本体HTMLの先頭にポータルへ戻るための最小限のナビゲーションリンクだけを追加しています。本文・フレームワーク内容は改変していません。

今後、既存HTMLを改変する必要がある場合は、以下を事前にこのREADMEへ記載してください。

- 変更が必要な理由
- 変更対象ファイル
- 変更範囲
- 著作権・引用・表現上の確認事項

## 法的配慮

- このサイトはリポジトリ内のHTMLファイルを紹介するポータルです。
- 各ページの著作権はそれぞれの作成者に帰属します。
- 無断転載ではなく、紹介・リンク・最小限の要約を目的としています。
- 既存HTMLの長文本文をトップページへ転載しない方針です。
- 他サイトから文章、画像、説明文は取得していません。
- 外部素材、外部フォント、CDN、有料素材は使用していません。
- 問題がある場合はIssueで連絡してください。

## 表現上の配慮

トップページでは、強い対立や操作を連想しやすい表現を避け、教育・思考整理・実務改善の文脈に言い換えています。

例：

- 形勢逆転 → 不利な状況の整理と打開
- 社内政治 → 組織内調整・意思決定支援
- 天才っぽく見せる → 知的に伝える表現技術

フレームワーク名そのものは既存HTMLへのリンク名として表示していますが、紹介文では中立的な説明にしています。

まず結論：勝つとは「相手を黙らせる」ではなく「意思決定条件を握る」こと

人間関係・社内政治・口喧嘩で本当に強い人は、声が大きい人でも、即興で言い返せる人でもありません。 本当に強いのは、何を勝利条件にするか、誰の前で戦うか、どの証拠を残すか、どのリスクを誰に選ばせるかを設計できる人です。

使用上の境界線： 嘘、捏造、脅迫、人格攻撃、ハラスメント、秘密の暴露、違法な録音・監視、報復は短期的に効いても中長期で自分の信用を破壊します。 このプレイブックは、相手を破壊するためではなく、不利な土俵から抜け、正当な優位を作るために使ってください。

24フレームワーク詳細

各カードは「概要」「構造」「具体例」「そのまま使えるセリフ」で構成しています。 検索・フィルタで状況に合うものだけを表示できます。

1勝利条件の再定義フレーム

攻勢・反転社内政治会議

不利な勝負は、相手に都合のよい勝利条件で始まっている。まず「何をもって勝ちか」を作り替える。

構造
  1. 現在の土俵を特定する。例：速さ、謝罪、個人責任、印象。
  2. 自分が勝てる土俵を設定する。例：リスク、再発防止、品質、顧客影響。
  3. 新しい土俵を「組織の利益」に接続する。
  4. 相手に反論しづらい選択肢として提示する。
具体例：「遅い」と詰められたら、速さだけでなく「品質事故リスク」「検証省略の影響」「再発防止」に議論を移す。
使うセリフ： 「納期が重要なのは同意です。ただ、ここで決めるべきなのは“誰が悪いか”ではなく、“納期・品質・再発防止のどのリスクを取るか”です。選択肢はA/B/Cです。」

2盤面マッピング・フレーム

社内政治防御

戦う相手本人だけを見るな。勝敗は決定者・影響者・観衆・記録の配置で決まる。

構造
  1. 決定者：最終的に判断する人。
  2. 影響者：決定者の認識を左右する人。
  3. 実務者：事実を持っている人。
  4. 観衆：あなたの評判を形成する人。
  5. 各人の利害・恐れ・欲しい成果を書く。
具体例：直属上司が不利な評価をしているなら、上司を論破するより、部長が気にする「品質リスク」「顧客影響」「再発防止策」を先に整理する。
使うセリフ： 「この件は関係者ごとに論点が違います。実務論点、品質論点、決裁論点を分けて整理し、判断者が決めやすい形にします。」

3三層の優位フレーム：事実・解釈・物語

攻勢・反転社内政治

人は事実だけでは動かない。事実、解釈、物語を揃えると、印象戦で逆転できる。

構造
  1. 事実：日付、数字、発言、成果物。
  2. 解釈：なぜ起きたか。どの構造が原因か。
  3. 物語：組織にとって何を守る話なのか。
具体例：個人ミス扱いされたら、「仕様変更遅れ」「レビュー不在」「承認待ち」を事実として出し、「個人の注意不足ではなく、変更管理プロセスの穴」と解釈し、「顧客品質を守る改善」に物語化する。
使うセリフ： 「事実としてはA、B、Cです。解釈としては個人の注意不足ではなく、変更管理の抜けが再現性を持って起きています。したがって、論点は再発防止です。」

4BATNA強化フレーム

交渉攻勢・反転

代替案がない人は交渉で弱い。別ルートを持つだけで、相手への依存度が下がり、態度が変わる。

構造
  1. 相手が協力しない場合の別案を作る。
  2. 別案のコスト・期限・責任範囲を明示する。
  3. 脅しではなく、合理的な選択肢として提示する。
具体例：他部署が情報を出さないなら、「協力ありなら3日短縮、なしなら推定で進めるが、インターフェース責任は先方に残る」と提示する。
使うセリフ： 「A案は貴部門の確認込みで安全に進める案、B案はこちら単独で進める案です。B案の場合、未確認部分のリスクは残ります。どちらで進めるのがよいでしょうか。」

5ZOPA設計フレーム

交渉会議

「無理です」では負けやすい。期限・範囲・品質・責任・承認手順を変数にして、交渉可能領域を作る。

構造
  1. 譲れない条件を明確にする。
  2. 譲れる条件を洗い出す。
  3. 相手の譲れない条件を仮説化する。
  4. 交換できる変数を増やす。
具体例：納期短縮を要求されたら、「範囲削減」「検証段階化」「承認簡略化」のどれを取るか選ばせる。
使うセリフ： 「納期短縮は検討できます。ただし同じ範囲・同じ品質保証・同じ承認手順では成立しません。範囲、検証、承認のどれを調整しますか。」

6アンカー先打ちフレーム

攻勢・反転交渉

最初に置かれた基準値は議論全体を支配する。相手の基準を待つな。自分に有利な比較軸を先に置く。

構造
  1. 過去案件・標準工数・規格・相場を探す。
  2. 妥当レンジを先に提示する。
  3. 相手の極端な要求は「その条件なら」と条件付きで扱う。
具体例：「見積もりが高い」と言われる前に、「過去案件では10〜15日、今回は追加要件で+3日、妥当レンジは13〜18日」と先に置く。
使うセリフ： 「まず基準を置きます。過去の同等案件はA〜B日です。今回は追加条件Cがあるため、妥当レンジはD〜E日です。」

7争点分解フレーム

防御会議攻勢・反転

強い言葉で押された時ほど、論点を分解する。抽象攻撃を具体論に落とすと相手の勢いは落ちる。

構造
  1. 事実：いつ、何が起きたか。
  2. 定義：何をもって悪い・遅い・不十分と言うのか。
  3. 因果：それが何を引き起こしたのか。
  4. 責任：誰のどの役割か。
  5. 提案：次に何を変えるか。
具体例：「お前の対応が悪い」と言われたら、「悪いの定義」「対象行動」「影響」「代替案」を順に聞く。
使うセリフ： 「整理させてください。“悪い”という評価を、事実・影響・次の対策に分けたいです。具体的に、いつのどの対応を指していますか。」

8価値交換フレーム

交渉社内政治

人は正論だけでは動かない。相手が欲しい評価・手柄・安心・時間・面子を通貨として扱う。

構造
  1. 相手が欲しいものを推定する。
  2. 自分が提供できるものを棚卸しする。
  3. Give→Getの順で交換する。
具体例：相手部署の協力が必要なら、「成果報告では相手部署の支援として明記する。その代わりレビューを優先してほしい」と提案する。
使うセリフ： 「この成果は貴部門の支援として明記します。その代わり、今回のレビューだけ優先的に入っていただけると助かります。」

9正当性の柱フレーム

防御攻勢・反転

「自分は正しい」では弱い。規範・データ・前例・権威のうち最低2本で主張を支える。

構造
  1. 規範：ルール、プロセス、規格、契約。
  2. データ：実験値、工数、発生率、ログ。
  3. 前例：過去案件、他部署事例。
  4. 権威：専門家、監査、顧客要求、責任者判断。
具体例：仕様変更を止めたいなら、「規格上の制約」「不具合率上昇データ」「過去に同条件で却下された前例」を合わせて出す。
使うセリフ： 「これは好みの問題ではありません。ルール上A、データ上B、過去事例上Cなので、このまま進めるのは妥当ではありません。」

10リスク反転フレーム

防御会議交渉

押し付けられた要求は、選択肢とリスクに変換して、決定者に選ばせる。これで責任の押し付けを防ぐ。

構造
  1. A/B/Cの選択肢にする。
  2. 各選択肢のメリット・リスク・責任を明示する。
  3. 「どれで決裁しますか」と聞く。
具体例：「テストを省いて出せ」と言われたら、省略案・段階導入案・通常検証案のリスクを比較する。
使うセリフ： 「Aは早いですが品質リスクが残ります。Bは2日遅れますがリスクを下げられます。Cは通常品質ですが納期は延びます。どのリスクを取る判断にしますか。」

11プロセス合意フレーム

会議防御

会議で不利になる原因は、議論内容ではなく「決め方」が曖昧なこと。先にプロセスを握る。

構造
  1. 今日の目的を言う。
  2. 論点を3つ以内に絞る。
  3. 決定者と決定条件を明確にする。
  4. 論点外・人格攻撃を止める基準を作る。
具体例：荒れそうな会議では冒頭で「今日は原因追及ではなく、再発防止策と決裁条件を決める」と宣言する。
使うセリフ： 「今日の目的はAを決めることです。論点は1、2、3に絞ります。個人評価ではなく、再発防止と次アクションに限定して進めます。」

12証拠の時系列フレーム

防御社内政治

記憶で戦うな。時系列で固定せよ。責任転嫁に最も効くのは、感情ではなく日付と記録である。

構造
  1. 日付、依頼、回答、未決、承認を並べる。
  2. 評価語を入れず、事実だけ書く。
  3. 関係者に共有し、共通認識にする。
  4. 次のアクションと担当を明記する。
具体例：「あなたが放置した」と言われたら、「○/○依頼、○/○回答待ち、○/○再確認、現在未承認」と1枚で示す。
使うセリフ： 「認識ズレを避けるため、事実ベースで時系列を整理します。○日に依頼、○日に確認、○日時点で未承認です。次の判断者は誰にしますか。」

13要約→同意→転換フレーム

防御会議

正面反論は相手の闘争心を上げる。一度受け止めてから、議論の向きを変える。

構造
  1. 要約：相手の懸念を短く言い直す。
  2. 同意：一部だけ同意する。
  3. 転換：だからこそ、次の選択肢に移る。
具体例：「遅い」と言われたら、「納期懸念ですね。重要です。だから範囲・検証・承認のどれを調整するか決めましょう」と返す。
使うセリフ： 「つまり懸念はAですね。その点は重要です。だからこそ、今はBかCのどちらで進めるか決める必要があります。」

14スティールマン反転フレーム

会議攻勢・反転

相手の主張を弱く解釈して叩くと泥沼になる。あえて強く解釈してから、条件不足を突く。

構造
  1. 相手の主張を最も合理的な形に言い換える。
  2. その主張が成立する条件を示す。
  3. 現在不足している条件や副作用を示す。
  4. より現実的な代替案に移す。
具体例：「全部急いでやれ」に対し、「最短で価値を出したい意図は正しい。ただし検証省略の条件では品質リスクが高い」と返す。
使うセリフ： 「ご意図は、最短で成果を出すことだと理解しています。それ自体は正しいです。ただ、その案が成立するにはA条件が必要で、現状は未達です。代替としてB案が現実的です。」

15ソクラテス質問フレーム

防御攻勢・反転

不利な場で無理に話すな。問う側に回れば、相手の根拠不足・定義不足・因果の飛躍が見える。

構造
  1. 定義質問：それは何を意味しますか。
  2. 根拠質問：何を根拠にしていますか。
  3. 例外質問：逆のケースはありますか。
  4. 影響質問：それをすると何が起きますか。
  5. 代替質問：他の選択肢はありますか。
具体例：「普通はこうする」と言われたら、「普通とはどの基準ですか」「過去案件ではどうでしたか」と聞く。
使うセリフ： 「確認です。その判断の基準は何でしょうか。根拠となるデータや前例はありますか。もし例外があるなら、今回がどちらに当たるか整理したいです。」

163文主張フレーム

会議防御

攻められている時ほど長く話すな。結論・根拠・次アクションの3文で主導権を取り戻す。

構造
  1. 結論：私は○○を提案します。
  2. 根拠：理由はA/Bです。
  3. 次：今日は××を決めたいです。
具体例：議論が荒れたら、「結論：段階導入を提案。理由：品質リスクを下げつつ納期影響を抑えるため。今日決めたいのは対象範囲」と返す。
使うセリフ： 「結論としてAを提案します。理由はBとCです。今日ここで決めたいのはDです。」

17面子保存の出口フレーム

社内政治交渉

相手を完全敗北させると、恨みと反撃が残る。勝つには、相手が降りられる理由を用意する。

構造
  1. 相手の過去判断を否定しない。
  2. 前提が変わったことにする。
  3. 相手の貢献や意図を認める。
  4. 新しい案への移行を自然にする。
具体例：相手案が間違いでも、「当時の情報では妥当だった。新データが出たので更新する」と言う。
使うセリフ： 「当時の前提ではその判断は合理的だったと思います。ただ、その後Aという情報が出たので、現時点ではB案に更新するのが安全です。」

18境界線フレーム

防御会議

人格攻撃・罵倒・圧力に乗った瞬間、相手の土俵で負ける。冷静に境界線を引く。

構造
  1. 観測：今起きた発言・行動を指摘する。
  2. 影響：議論に与える悪影響を言う。
  3. 要求：事実と論点に戻す。
  4. 次：続く場合の対応を示す。
具体例：「無能」と言われたら、「その表現では議論が進まない。事実と対策に戻したい」と返す。
使うセリフ： 「今の表現は人格評価に聞こえます。その形だと議論が前に進みません。事実、影響、次の対策に戻してください。続く場合は、上位者同席で再設定します。」

19観衆設計フレーム

社内政治防御

誰の前で戦うかで勝敗は変わる。勝てない場で即反撃するより、場を再設計する方が強い。

構造
  1. 1on1で解くべきか判断する。
  2. 第三者同席が必要か判断する。
  3. 議事録・決定者・専門家を入れるか判断する。
  4. 公開範囲を広げるリスクを読む。
具体例：会議で吊し上げられたら、その場で感情反論せず、「論点整理して関係者同席で再設定」と提案する。
使うセリフ： 「この場で印象論のまま進めると誤解が残ります。論点と事実を整理したうえで、関係者同席で再度決めたいです。」

20サラミ合意フレーム

攻勢・反転社内政治

大きな合意をいきなり取りに行くと潰される。小さなYESを積み上げ、最後に自然な帰結にする。

構造
  1. 大目標を小さな合意に分割する。
  2. 反対されにくい順に並べる。
  3. 合意を記録する。
  4. 最終提案を既合意の延長にする。
具体例：改善提案を通したいなら、問題認識→データ取得→小規模実験→効果確認→展開の順に進める。
使うセリフ： 「まず大きな導入を決める必要はありません。問題認識の合意、データ取得の合意、小規模検証の合意まで進めたいです。」

21反対意見先取りフレーム

攻勢・反転会議

反論されてから防御すると遅い。想定反論を自分から出し、処理済みにする。

構造
  1. 相手が言いそうな反対を3つ書く。
  2. それぞれに短い回答を用意する。
  3. 会議では自分から「懸念は3つあります」と言う。
具体例：提案前に「コスト、時間、責任範囲が懸念だと思うので先に整理します」と言う。
使うセリフ： 「この提案への懸念は、おそらくコスト、時間、責任範囲の3点です。先にそれぞれの対策を示します。」

22タイミング支配フレーム

社内政治攻勢・反転

同じ内容でも、出す順番とタイミングで結果は変わる。政治は内容だけでなく、順序のゲームである。

構造
  1. サプライズで出すか、事前根回しするか選ぶ。
  2. 決定者の関心が高いタイミングを狙う。
  3. 会議序盤で基準を置く。
  4. 疲労した終盤の雑決定を避ける。
具体例：会議終盤に重要論点を詰められたら、「次回冒頭で10分確保」と返す。
使うセリフ： 「これは重要な判断なので、会議終盤の残り時間で決めるのは危険です。次回の冒頭で10分確保して、選択肢とリスクを出します。」

23責任分解フレーム

防御社内政治

責任が曖昧だと全部あなたのせいになる。決める人・実行する人・助言する人・共有される人を分ける。

構造
  1. D：Decision。誰が決めるか。
  2. E：Execute。誰が実行するか。
  3. C：Consult。誰に相談するか。
  4. I：Inform。誰に共有するか。
具体例：トラブル時に「実装は私、仕様判断はAさん、レビューはBさん、承認はCさん」と分解する。
使うセリフ： 「責任を混ぜると再発します。決裁、実行、レビュー、共有を分けます。今回の未決はどの役割で止まっていたか確認したいです。」

24エスカレーション・ラダー

防御社内政治

上にあげるかどうかを感情で決めると負ける。段階、条件、記録、撤退線を先に決める。

構造
  1. 1on1で整理する。
  2. 小会議と議事録で固定する。
  3. 決定権者を同席させる。
  4. 正式プロセスに乗せる。
  5. これ以上は損という撤退線を持つ。
具体例：合意が何度も反故にされるなら、2回目で議事録、3回目で上位者同席、それでも無理なら正式ルートへ。
使うセリフ： 「同じ認識ズレが繰り返されているため、次回から議事録を残します。それでも判断が進まない場合は、決定権者同席で再設定します。」

実戦テンプレート集

1. 会議後の記録固定メール

件名：本日の論点整理と次アクション 本日の議論について、認識ズレ防止のため整理します。 1. 決定事項 - ○○ 2. 未決事項 - ○○：判断者は○○さん - ○○：追加情報は○/○までに確認 3. リスク - A案：メリット○○ / リスク○○ - B案：メリット○○ / リスク○○ 4. 次アクション - ○○：担当○○、期限○/○ 認識違いがあればご指摘ください。

2. 責任転嫁された時の一言

「個人責任の話にする前に、時系列・役割・判断ポイントを分けたいです。事実としては、○日にA、○日にB、○日時点でCです。」

3. 強い要求を受けた時の一言

「実現可能性はあります。ただし、同じ納期・同じ範囲・同じ品質保証を同時に満たすのは難しいです。どの条件を調整するか決めたいです。」

4. 罵倒・人格攻撃を受けた時の一言

「その表現では議論が前に進みません。事実、影響、次の対策に戻してください。続く場合は、場を改めます。」

5. 相手に逃げ道を与えつつ逆転する一言

「当時の前提ではその判断は合理的でした。ただ、新しい情報が出たので、現時点では判断を更新するのが安全です。」

最後に：一番危険な負け筋

最も危険なのは、相手に腹が立った瞬間に「言い返すこと」を勝利条件にしてしまうことです。 口では勝っても、観衆・記録・評価・決定権者を失えば、実務では負けです。

強い人は、怒りを否定しません。しかし怒りをそのまま出さず、証拠、選択肢、場、時間、記録に変換します。 この変換能力こそが、社内政治・人間関係・口論での本質的な戦闘力です。
